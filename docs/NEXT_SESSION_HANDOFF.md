## 최종 목표

지금까지의 기획 대화와 `docs` 폴더의 문서를 기반으로, 마비노기 쿨타임 오버레이 MVP를 제작합니다.

MVP의 최소 구현 목표는 다음과 같습니다.

1. WGC로 게임 창을 확인합니다.
2. 마비노기 게임임이 확인되면 캡쳐합니다.
3. 캡쳐 화면에서 자동으로 퀵슬롯 후보를 선택합니다.
4. 사용자에게 후보 슬롯을 보여주고, 사용할 슬롯만 체크하게 합니다.
5. 사용자가 원하는 크기의 캔버스를 만들고, 선택한 슬롯들을 그 캔버스에 자유롭게 배치합니다.
6. 배치된 캔버스를 화면 내 적절한 위치로 옮기고 투명도를 조절합니다.
7. 시작 버튼을 누르면 항상 위에 표시되는 오버레이가 출력됩니다.
8. 오버레이는 마우스 클릭을 먹으면 안 됩니다.
9. 설정 가능한 키 조합 또는 베이스 프로그램 UI의 중지 버튼으로 오버레이를 중지할 수 있어야 합니다.
10. 베이스 프로그램 UI는 계속 떠 있을 수 있지만 항상 위 창이어서는 안 됩니다.

## 사용자가 직접 지적한 현재 문제

다음 항목은 아직 사용자의 실제 수동 테스트 기준으로 충분히 검증되지 않았거나, 추가 개선 여지가 큽니다.

- 현재 UI 디자인이 엉망이라는 피드백이 있었습니다.
- 아이콘 인식률이 낮고, 슬롯이 아닌 부분까지 후보로 잡는 문제가 있었습니다.
- 모든 스킬 아이콘은 같은 크기인데, 과거 구현에서 후보 사각형 크기가 제각각 나오는 문제가 있었습니다.
- 현재 컴퓨터 환경의 슬롯 크기는 29px입니다.
- 슬롯 크기 조절은 2px 단위가 아니라 더 세밀해야 한다는 피드백이 있었습니다. 현재는 1px 단위로 바뀌어 있습니다.
- 퀵슬롯은 절대로 겹치지 않는다는 조건을 감지 로직에 반영해야 합니다. 현재 겹침 억제는 들어갔지만 사용자 테스트는 아직 필요합니다.
- 하단에는 퀵슬롯이 절대로 오지 않는다고 했습니다. 현재 하단 감지는 제외되어 있습니다.
- 지정 후 시작을 누르면 프로그램이 꺼지는 문제가 있었습니다. 이후 예외 로깅과 시작 try/catch를 보강했지만, 런타임 재검증은 사용자가 하기로 했습니다.

## 다음 구현 후보

우선순위는 다음과 같습니다.

1. UI 정리
   - 현재 메인 UI가 기능 위주로 누적되어 있으므로, 단계별 워크플로우를 더 명확히 나눕니다.

   - 버튼 이름과 상태 문구를 실제 사용 흐름에 맞게 다듬습니다.

2. 감지 로직 안정화
   - 현재 감지는 슬롯 프레임/아이콘 대비를 기준으로 후보를 잡고, 이후 겹침 억제를 적용하는 방식입니다.
   - 다음 개선은 “단일 슬롯 크기 기반의 격자 후보 확장”이 더 적절합니다.
   - 한 슬롯을 찾으면 주변에 같은 간격의 슬롯이 있을 가능성을 추정해서 행/열 후보를 확장하고, 겹침 없이 정렬된 후보만 남기는 방식이 좋습니다.
   - false positive를 줄이려면 랜덤한 단일 박스보다 “동일 크기, 동일 간격, 행/열 연속성”을 더 높은 점수로 둬야 합니다.

3. 레이아웃 편집 사용성 개선
   - 배치 슬롯 클릭 선택 상태를 더 명확히 표시합니다.
   - 키보드 Delete 처리 범위를 레이아웃 편집 창에도 안정적으로 적용합니다.
   - 배치 슬롯을 캔버스 밖으로 끌고 나가지 못하게 clamp합니다.
   - 캔버스 크기 변경 시 기존 슬롯 위치를 유지하거나 경계 안으로 보정합니다.

4. 오버레이 시작 안정성
   - 사용자가 제공하는 `%APPDATA%\TestOverlayProj\Logs\app.log`를 기준으로 원인을 확인합니다.
   - 이전에는 시작 버튼 이후 `Overlay started` 로그가 남지 않아 오버레이 생성/표시 근처 예외로 추정했습니다.
   - 이미 try/catch와 전역 예외 로깅을 넣었으므로, 다음 오류는 로그 기반으로 좁혀야 합니다.

5. MVP 수동 검증 문서 갱신
   - 사용자가 직접 테스트한 결과를 반영해 `docs\MVP_MANUAL_TEST.md`에 체크리스트를 갱신합니다.
   - 자동 테스트처럼 표현하지 말고, 사용자 수동 검증 기준으로 유지합니다.

## 감지 로직 관련 현재 정책

현재 정책은 다음 방향으로 맞춰져 있습니다.

- 퀵슬롯은 같은 크기라고 봅니다.
